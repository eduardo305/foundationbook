Introduction

What is Foundation?

Foundation is a responsive front-end framework. Foundation provides a responsive grid and HTML and CSS UI components, templates, and code snippets, including typography, forms, buttons, navigation and other interface components, as well as optional JavaScript extensions. Foundation is maintained by zurb.com and is an open source project.

In this material, you will find the basics of Foundation framework. We're goind to walk through the main CSS and JavaScript components offered. However we'll give an extra focus on the grid system, which is the main tool offered here.

Why Foundation?

Is it better than Bootstrap?

The answer here is: it depends. Foundation is so strong and complete as Bootstrap is. Each one of these 2 frameworks can offer you a good support when developing/prototyping your website. The main feature that Foundation offer is its grid which was the first responsive system available on the internet. That's why it is so recognizable. In other hands, Bootstrap team made a good work and they could also develop a powerful grid system.

The main advantage Bootstrap presents over Foundation is its bigger community. We can find a considerably higher number of templates and components built by third-parties.

The bottomline is: the frontend framework choice is personal. Each one likes more one over another. A good start point is to decide which components you're going to use on your website and see which one of the two frameworks portfolios better match your needs. If you only need a grid system, maybe the best choice is Foundation.

Below are 2 good links that already made a full comparison between them:

  • http://responsive.vermilion.com/compare.php
  • http://www.sitepoint.com/grid-system-comparison-bootstrap-vs-foundation/

Getting Started

Basic setup

On the Foundation website you can find a variety of ways to install Foundation. The good point here is that Foundation allows you to customize your installation by choosing only what you need in your website. On our course, we'll use either bower way or the cdnjs way. If you want more details, please check the documentation. The link is http://foundation.zurb.com/develop/download.html

Setup way 1

The bower way

Bower is a package management system for client-side programming on the World Wide Web. It depends on Node.js and npm. It works with git and GitHub repositories. In our course, we are assuming you already have it installed in your computer, but if you don't please check its documentation on internet

bower install foundation --save
Code formated by http://hilite.me/

Now that you have all Foundation packages installed on your application, you are only suppose to add them on your index.html as you would do for any other script.

Setup way 2

The cdnjs way

If you need a quick prototyping, you can make use of a external scripts repository, such as CDNJS. Navigate to http://cdnjs.com/libraries/foundation and make a copy of foundation.min.css. Now all you need to do is to include it on your index.html. In this example, we are only adding the css file, because all we need is grid system. Later we are going to deal with a few Javascript components as well.

<script  type="text/javascript" src="https://cdnjs.cloudflare.com/ajax/libs/foundation/5.5.2/css/foundation.min.css"></script>
Code formated by http://hilite.me/

Hands on

The grid system

The grids system is the tool that allow you create powerful multi-device layouts quickly and easily with the default 12-column, nest-able Foundation grid. If you're familiar with grid systems, you'll feel right at home. If not, you'll learn quickly.

The grid works by defining rows (horizontal cuts) and columns (vertical cuts). We can have, by default, 12 columns inside a row, however this number is configurable. We have als pre-defined classes for each kind of screensize. This is what makes the magic here. We can have a perfect responsive website layout just by defining the classes you are going to use. For smartphones, we have small. For tablets we have medium. And for desktops we have large. Below we can see the code for the grid displayed on this section:

Block grid

Evenly aligning your website

Block grids give you a way to evenly split contents of a list within the grid. If you wanted to create a row of five images or paragraphs that need to stay evenly spaced no matter the screen size, the block grid is for you. 

<ul class="small-block-grid-3">
  <li><!-- Your content goes here --></li>
  <li><!-- Your content goes here --></li>
  <li><!-- Your content goes here --></li>
</ul>

Visibility

Is this to be shown for which screen size?

Foundation gives you the ability to show or hide components from your page based on your screen size. The framework offers lots of helper classes to help you to decide where to change it. The classes names are very clear and are shown below: 

<p class="panel">
  <strong class="show-for-small-only">This text is shown only on a small screen.</strong>
  <strong class="show-for-medium-up">This text is shown on medium screens and up.</strong>
  <strong class="show-for-medium-only">This text is shown only on a medium screen.</strong>
  <strong class="show-for-large-up">This text is shown on large screens and up.</strong>
  <strong class="show-for-large-only">This text is shown only on a large screen.</strong>
  <strong class="show-for-xlarge-up">This text is shown on xlarge screens and up.</strong>
  <strong class="show-for-xlarge-only">This text is shown only on an xlarge screen.</strong>
  <strong class="show-for-xxlarge-up">This text is shown on xxlarge screens and up.</strong>
</p>
The above script shows you how to show a component depending on your screen size. In case you want to hide something, you can make use of the classes shown below: 
<p class="panel">
  <strong class="hide-for-small-only">You are <em>not</em> on a small screen.</strong>
  <strong class="hide-for-medium-up">You are <em>not</em> on a medium, large, xlarge, or xxlarge screen.</strong>
  <strong class="hide-for-medium-only">You are <em>not</em> on a medium screen.</strong>
  <strong class="hide-for-large-up">You are <em>not</em> on a large, xlarge, or xxlarge screen.</strong>
  <strong class="hide-for-large-only">You are <em>not</em> on a large screen.</strong>
  <strong class="hide-for-xlarge-up">You are <em>not</em> on an xlarge screen and up.</strong>
  <strong class="hide-for-xlarge-only">You are <em>not</em> on an xlarge screen.</strong>
  <strong class="hide-for-xxlarge-up">You are <em>not</em> on an xxlarge screen.</strong>
</p>
Foundation also provides a way to detect if the device orientation you are currently using. You can define classes to specify if something is going to display only on landscape or portrait orientation. Isn't it handy? 
<p class="panel">
  <strong class="show-for-landscape">You are in landscape orientation.</strong>
  <strong class="show-for-portrait">You are in portrait orientation.</strong>
</p>

Buttons

Just push the button

Buttons are very convenient user insterface components used since the beginnings on the web. Foundation already offers a default style for them. Of course, you can customize them whenever you need. The minimal markup for a button is: 

<a href="#" class="button">Default Button</a>
This is how you are going to render the default Foundation button on your page. Although we are using 'a' component, nothing stops us to use the 'button' html component. It will also render a special button for you. Besides, it's look & feel, Foundation also supports different button sizes, such as tiny, small, large or expanded. You can also add a class 'disabled' on it, and it will display as it is disabled. Some helper classes can be used along with, so you define it as rounded button for example. Below are a few classes applied for buttons: 
<!-- Size Classes -->
<a href="#" class="button tiny">Tiny Button</a>
<a href="#" class="button small">Small Button</a>
<a href="#" class="button">Default Button</a>
<a href="#" class="button disabled">Disabled Button</a>
<a href="#" class="button large">Large Button</a>
<a href="#" class="button expand">Expanded Button</a>
<!-- Radius Classes -->
<a href="#" class="button round">Round Button</a>
<a href="#" class="button radius">Radius Button</a>
You can also change the button colors easily. Foundation provides a few classes for standard states, such as success, alert, info, among others: 
<!-- Color Classes -->
<a href="#" class="button">Default Button</a>
<a href="#" class="button success">Success Button</a>
<a href="#" class="button secondary">Secondary Button</a>
<a href="#" class="button alert">Alert Button</a>
<a href="#" class="button info">Info Button</a>
<a href="#" class="button disabled">Disabled Button</a>

Floating, radius, rounding and aligning

Lots of times we need to float and/or align components on the page. Foundation is very handy on this part. Lots of classes are offered to make your life easier. Forms

Components and more components

It was never so simple to create a layout for a form like we can do now. Foundation mixes form tags with its grid system to produce powerful form layouts. A good approach is to create rows inside your form tag.
The below code will result in the layout displayed above: 

<form action="">
           <div class="row">
             <div class="small-12 columns">
               <label for=""> 12 columns full width
                 <input type="text" placeholder="12 columns">
               </label>
             </div>
           </div>
           
           <div class="row">
             <div class="small-12 columns">
               <label for="">12 columsn - select full width
                 <select name="" id="">
                   <option value="1">1</option>
                   <option value="2">2</option>
                   <option value="3">3</option>
                 </select>
               </label>
             </div>
           </div>
           
           <div class="row collapse">

               <label for="">prefix</label>
               <div class="small-3 columns"><span class="prefix">http://</span></div>
               <div class="small-9 columns">
                 <input type="text" placeholder="prefix input">
               </div>

           </div>
           
           <div class="row collapse">
             <div class="small-9 columns">
               <input type="text" placeholder="postfix input">
             </div>
             <div class="small-3 columns">
               <a class="button postfix">go</a>
             </div>
           </div>
         </form>
